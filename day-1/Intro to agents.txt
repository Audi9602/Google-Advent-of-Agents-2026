Core Concepts and Definitions
Paradigm Shift: AI is evolving from predictive models that require human direction for every step to autonomous agents capable of independent problem-solving.
Definition of an Agent: An agent is a complete application that uses a Language Model (LM) in a continuous loop with tools to accomplish an objective.
Core Anatomy:
Model (The Brain): The central reasoning engine.
Tools (The Hands): Mechanisms that connect the agent to the outside world for actions and data.
Orchestration (The Nervous System): The governing layer that manages planning, memory, and strategy.
Deployment (The Body/Legs): The infrastructure required to run the agent as a reliable service.
The Agentic Problem-Solving Process
Agents operate in a 5-step cyclical loop:
Get the Mission: Receive a high-level goal.
Scan the Scene: Perceive the environment and gather context.
Think It Through: Devise a multi-step plan using the reasoning model.
Take Action: Execute concrete steps (e.g., calling an API).
Observe and Iterate: Incorporate results into memory and repeat the cycle until the mission is complete.
Taxonomy of Agentic Systems (Levels 0-4)
Level 0 (Core Reasoning): An isolated model with no tools or real-time awareness.
Level 1 (Connected Problem-Solver): Uses tools (like Google Search or RAG) to access live data.
Level 2 (Strategic Problem-Solver): Employs context engineering to manage complex, multi-part goals.
Level 3 (Collaborative Multi-Agent): A "team of specialists" where agents delegate tasks to other agents.
Level 4 (Self-Evolving System): Highly autonomous systems that can dynamically create new tools or agents to fill capability gaps.
Architectural and Design Choices
Model Selection: Success depends on finding the optimal intersection of quality, speed, and cost for specific tasks rather than generic benchmarks.
Tooling Types:
Grounding: Using RAG or NL2SQL to reduce hallucinations with factual data.
Action: Executing code or calling APIs to change the world.
Human-in-the-Loop (HITL): Tools for pausing workflows for human confirmation.
Orchestration Design: Developers act as "directors" rather than "bricklayers," using system prompts to define an agent's "constitution" and persona.
Memory Management:
Short-term: An active scratchpad of current conversation history.
Long-term: Persistent storage (often vector databases) allowing agents to remember user preferences over weeks.
Multi-Agent Patterns: Common designs include the Coordinator (manager/specialist routing), Sequential (linear assembly line), and Iterative Refinement (generator/critic loops).
Agent Ops: Production Management
Shift from Deterministic to Stochastic: Testing requires moving from "pass/fail" unit tests to probabilistic quality evaluations.
Metrics: Success is measured via real-world KPIs like goal completion rates and ROI.
LM as Judge: Powerful models are used to evaluate agent outputs against a predefined rubric.
Debugging: OpenTelemetry traces provide a "high-fidelity" recording of an agent's reasoning trajectory to diagnose issues.
Human Feedback: "Thumbs down" or bug reports are treated as valuable edge cases to improve automated evaluation datasets.
Interoperability and the Agentic Economy
Agent-Human Interaction: Ranges from text-based chatbots to multimodal "live mode" where agents see and hear in real-time.
Agent-to-Agent (A2A): An open protocol using Agent Cards to allow specialized agents to discover and collaborate with each other.
Agents and Money: Protocols like AP2 (cryptographic mandates) and x402 (micropayments) establish trust for autonomous financial transactions.
Security and Governance
Defense-in-Depth: Security requires a hybrid of deterministic hardcoded guardrails and AI-powered "guard models".
Agent Identity: Agents are treated as a new category of "principal" requiring digital passports (e.g., SPIFFE) for granular access control.
Enterprise Control Plane: A central gateway and registry are used to prevent "agent sprawl" by managing all agentic traffic, policies, and discovery.
Learning and Evolution
Self-Evolution: Agents learn from runtime experience (logs/traces) and external signals (policy changes) to optimize their own prompts and tools.
Agent Gym: The "next frontier" involving offline simulation environments to pressure-test agents with synthetic data and trial-and-error optimization.
Advanced Use Cases
Google Co-Scientist: A multi-agent ecosystem that acts as a virtual collaborator to generate and evaluate scientific hypotheses.
AlphaEvolve: An agentic system that uses evolutionary code generation to discover and optimize complex algorithms.
